Это переработанная версия задачи

Что изменено по сравнению с оригиналом
Код стал принимать и `iterable` (чтобы можно было легко подменить `json_decode` на потоковый парсер).
Добавлена строгая проверка структуры: прежде чем декодировать base64, проверяется наличие `image_name` и `image.base64`.
Имена файлов теперь уникализируются хешем по сочетанию `image_name|product_name|link|timestamp`, поэтому файлы не перезаписываются и легко соотносятся с продуктом.
Запись ведётся в бинарном режиме и код аккуратно обрабатывает ошибки записи (файл не создаётся при невалидном base64).
